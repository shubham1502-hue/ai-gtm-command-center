# Contributing

Contributions should make this repo easier for founders and operators to fork, understand, run, or adapt.

## Useful contributions

- Clearer sample data or templates.
- Better input and output documentation.
- Validation checks for generated outputs.
- Optional integrations that do not make paid APIs mandatory.
- Founder-facing examples that show the decision artifact before the tooling.

## Contribution rules

- Do not commit secrets or private company data.
- Do not add real customer, prospect, borrower, merchant, investor, employee, or payment data.
- Keep paid APIs optional.
- Keep claims grounded in the repo contents.
- Prefer practical operating workflows over cosmetic changes.
- Keep pull requests small and tied to one issue or one clear improvement.
- Do not replace working scripts, templates, or docs with placeholders.
- Do not rewrite broad sections of the repo unless the issue explicitly asks for it.
- If you use AI tools, review the output yourself and remove generic, duplicated, or untested changes before opening a PR.

## Before opening a PR

- Run the smallest relevant test, script, or validation command you can.
- Run `git diff --check`.
- Mention exactly what you ran in the PR description.
- Explain what a founder or operator can do after your change.
